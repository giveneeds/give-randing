# /인수인계

현재 작업 맥락을 handoff 문서에 정리하고, 검증 후 관련 변경을 커밋한다.

## 사용 예

```text
/인수인계
/인수인계 콘텐츠 스튜디오 기둥 기반 주제 발굴 로직 정리
```

## 실행 절차

1. 루트의 `HANDOFF_GUIDE.md`와 최신 작업 전용 handoff 파일을 먼저 읽는다.
   - 콘텐츠 자동화 작업이면 `HANDOFF_CONTENT_AUTOMATION_2026-05-26.md`를 우선 갱신한다.
   - 작업 전용 handoff가 없으면 `HANDOFF_<AREA>_<YYYY-MM-DD>.md` 형식으로 새로 만든다.

2. `git status --short`와 필요한 `git diff`를 확인한다.
   - 사용자가 만든 unrelated 변경을 임의로 되돌리지 않는다.
   - 커밋에 포함할 파일과 제외할 파일을 판단한다.
   - DB/seed/migration 관련 변경이 있으면 데이터 무결성 위험을 먼저 설명한다.

3. handoff에 반드시 기록한다.
   - 이번 작업의 목표
   - 확정된 의사결정
   - 구현된 파일/기능
   - 검증 결과
   - 아직 남은 리스크
   - 다음 작업자가 바로 이어서 볼 체크포인트

4. 검증한다.
   - Next.js 변경이면 기본적으로 `npm run build`를 실행한다.
   - 빌드를 못 돌렸다면 handoff와 최종 보고에 이유를 남긴다.

5. 커밋한다.
   - 먼저 `git status --short`를 다시 확인한다.
   - 관련 파일만 `git add` 한다. 무조건 `git add -A`를 쓰지 않는다.
   - 커밋 메시지는 인자가 있으면 그 내용을 반영하고, 없으면 다음 형식을 쓴다.

```bash
git commit -m "chore: update handoff and commit current work"
```

6. 최종 보고한다.
   - handoff 파일 경로
   - 커밋 해시
   - 포함 파일 요약
   - 검증 결과
   - 제외한 변경 또는 남은 리스크

## 주의

- 배포는 이 명령의 기본 범위가 아니다. 사용자가 명시적으로 "배포까지"라고 말할 때만 push/deploy를 진행한다.
- 큰 작업 중간에 실행할 경우, 미완성 변경은 커밋하지 말고 handoff에만 남기는 선택을 할 수 있다.
- Supabase 데이터 변경은 반드시 SELECT precheck와 백업 후 진행한다.
