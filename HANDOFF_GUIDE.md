# 🚀 GIVENEEDS 마케팅 플랫폼 인수인계 가이드

본 문서는 기브니즈(Giveneeds) 프로젝트의 **1단계: 마케팅 통합 엔진 구축** 작업을 정리한 문서입니다. 향후 다른 AI 세션이나 개발자가 작업을 이어받을 때 이 문서를 가장 먼저 참조하십시오.

## 📅 1. 요구사항 타임라인 (Project Evolution)
- **최초 요구**: 단순 랜딩 페이지의 디자인 고도화 및 성능 개선 요청.
- **방향 선회**: 단순 웹사이트를 넘어, 광고 캠페인마다 다른 LP를 즉시 생성할 수 있는 **마케팅 플랫폼**으로 확장 결정.
- **핵심 UI**: 
  - 메인은 **'매거진 B'** 스타일의 미니멀하고 고급스러운 저널 레이아웃 적용. 
  - 모바일에서도 빠른 로딩과 고화질 이미지 유지 필수.
- **전략 포인트**: 
  - '리드 마그넷'(가이드북, 분석 리포트)을 활용한 고객 DB 수집 극대화. 
  - 프리미엄 콘텐츠는 신청 폼을 작성해야만 열리는 **Gating** 시스템 적용.

## 📌 2. 프로젝트 요약
기존의 단순 랜딩 페이지를 **[브랜드 매거진]**과 **[멀티 캠페인 랜딩페이지(LP)]**가 통합된 마케팅 플랫폼으로 전환하였습니다.

- **핵심 목표**: 광고 유입 경로별로 최적화된 무한 LP 생성, 브랜드 신뢰도를 높이는 매거진 발행, 고객 리드 데이터 통합 수집.
- **기술 스택**: Next.js (App Router), Supabase (Auth/DB), Tailwind CSS (Vanilla), Lucide Icons.

## 🛠️ 2. 아키텍처 및 라우팅
- **`/` (Home)**: '매거진 B' 스타일의 브랜드 지향적 에디토리얼 페이지. (`app/page.js`)
- **`/lp/[slug]` (Campaign LP)**: 광고 캠페인별 동적 랜딩 페이지. (`app/lp/[slug]/page.js`)
- **`/magazine/[slug]` (Magazine Detail)**: 아티클 상세 페이지 및 프리미엄 콘텐츠 잠금(Gating) 기능.
- **`/admin` (Dashboard)**: 프로젝트 통합 관리자 (캠페인 설정, 매거진 발행, 리드 조회).

## ✅ 3. 주요 구현 기능 (현재 상태)
- [x] **하이브리드 히어로 섹션**: 캠페인 성격에 따라 '파티클 텍스트(A형)' 또는 '리드폼/가이드북(B형)' 선택 가능.
- [x] **AI 전략 챗봇**: 사이트 전체에 부착된 AI 마케팅 진단 모듈 UI 및 시뮬레이션 로직.
- [x] **Lead Capturing (Gating)**: 프리미엄 아티클 열람 시 또는 LP에서 카카오/직접 입력을 통한 고객 데이터 수집 폼 연동.
- [x] **글로벌 섹션 시스템**: 공통 섹션(회사 소개, 프로세스 등)을 관리자에서 선택하여 개별 LP에 조합 가능.

## 📁 4. 핵심 파일 안내
- `/lib/supabase.js`: Supabase 클라이언트 및 데이터 스키마 정의 (더미/실서버 스위칭 지원).
- `/components/landing/`: 메인 및 LP용 UI 컴포넌트 모음.
- `/components/ui/LeadForm.js`: 전천후 리드 수집 및 카카오 로그인 UI.
- `/components/ui/AiChatbot.js`: 플로팅 AI 채팅 인터페이스.
- `database_schema.sql`: 실서버 Supabase 구축을 위한 SQL 스크립트.
