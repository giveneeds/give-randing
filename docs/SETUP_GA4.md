## 1부 — GA4 속성 생성

GA4(Google Analytics 4)는 구글이 제공하는 무료 웹 분석 도구입니다. 사이트마다 **측정 ID**(`G-` 로 시작)를 발급받아 코드에 심으면, 방문자 행동이 자동으로 기록됩니다.

### 1.1 구글 계정 준비

- 구글 계정으로 [https://analytics.google.com](https://analytics.google.com) 접속.
- 회사 운영 계정(gmail 또는 워크스페이스) 사용 권장. 개인 계정으로 만들면 나중에 권한 이전이 번거롭습니다.

### 1.2 계정(Account) 생성

GA4의 계층은 **계정 > 속성 > 데이터 스트림**입니다.

1. 첫 진입 시 **계정 만들기** 버튼.
2. 계정 이름: `GIVENEEDS` (회사 단위 이름. 나중에 매거진·캠페인용 속성 여러 개를 한 계정에 묶을 수 있음).
3. 계정 데이터 공유 설정은 모두 체크해도 됩니다(구글 익명 통계용, 한국 운영엔 영향 없음).
4. **다음**.

### 1.3 속성(Property) 생성

1. 속성 이름: `Giveneeds Main` (사이트 단위 이름).
2. 보고 시간대: **대한민국 (GMT+9)**.
3. 통화: **대한민국 원 (KRW)**.
4. **다음**.

### 1.4 비즈니스 세부정보

- 업종: 마케팅·광고
- 비즈니스 규모: 작음 (해당)
- 사용 목적: 모두 체크 (리드 측정 / 콘텐츠 측정 / 캠페인 측정 / 사용자 행동 분석)
- **다음**.

### 1.5 데이터 스트림 생성

플랫폼 선택: **웹**

- 웹사이트 URL: `https://www.giveneeds.co.kr`
- 스트림 이름: `Giveneeds Web`
- **향상된 측정** 토글: ON (기본). 페이지 스크롤·외부 링크 클릭 등을 자동 수집해줍니다.
- **스트림 만들기** 클릭.

### 1.6 측정 ID 복사

스트림 생성 직후 페이지 상단에 다음 형식의 ID가 표시됩니다:

```
G-XXXXXXXXXX
```

이 값을 복사해두세요. 다음 단계에서 환경변수로 넣습니다.

> 💡 측정 ID를 분실해도 GA4 → 관리 → 데이터 스트림 → 웹 스트림 클릭으로 언제든 다시 확인할 수 있습니다.

## 2부 — 환경변수 설치

### 2.1 로컬 개발 환경

프로젝트 루트의 `.env.local` 파일을 열고 다음 한 줄 추가:

```bash
NEXT_PUBLIC_GA_MEASUREMENT_ID=G-XXXXXXXXXX
```

`G-XXXXXXXXXX` 부분을 1.6에서 복사한 실제 ID로 교체.

> ⚠️ **반드시 `NEXT_PUBLIC_` 접두사가 필요합니다.** Next.js는 이 접두사가 있는 환경변수만 브라우저에 노출시킵니다. 접두사 빠지면 GA가 동작하지 않습니다.

추가 후 개발 서버 재시작:

```bash
npm run dev
```

### 2.2 운영(Vercel) 환경

1. [Vercel 대시보드](https://vercel.com) → 프로젝트 선택 → **Settings** → **Environment Variables**.
2. **Add New** 클릭.
3. Name: `NEXT_PUBLIC_GA_MEASUREMENT_ID`
4. Value: `G-XXXXXXXXXX`
5. Environment: **Production** + **Preview** 둘 다 체크 (스테이징도 측정하면 데이터가 섞이므로 Production만 체크하는 게 일반적이지만, 처음엔 둘 다 체크해서 동작 확인).
6. **Save**.
7. 다음 배포부터 자동 적용. 즉시 적용하려면 Vercel에서 **Redeploy** 클릭.

## 3부 — 동작 확인

### 3.1 실시간 보고서

1. [GA4 대시보드](https://analytics.google.com) → 보고서 → **실시간**.
2. 새 탭에서 `https://www.giveneeds.co.kr` 접속.
3. 30초 이내에 GA4 실시간 보고서에 **활성 사용자 1명**이 표시되면 정상.

표시 안 되면 [4.3 트러블슈팅](#43-트러블슈팅) 참조.

### 3.2 이벤트 확인

페이지 이동·CTA 클릭·폼 제출 시 GA4 실시간 보고서 우측의 **이벤트** 카드에 즉시 잡힙니다.

기브니즈 사이트는 다음 이벤트를 자동 전송합니다(자세한 목록은 4부 참고):

- `page_view` — 페이지 이동 시 자동
- `lead_form_view` — 리드 폼 노출
- `lead_form_submit` — 리드 폼 제출 성공
- `kakao_login_start` — 카카오 로그인 버튼 클릭
- `cta_click` — 일반 CTA 버튼 클릭

### 3.3 트러블슈팅

| 증상 | 가능성 높은 원인 | 해결 |
|------|------------------|------|
| 실시간 보고서에 안 잡힘 | 환경변수 누락 또는 오타 | `.env.local` / Vercel 환경변수 재확인. 접두사 `NEXT_PUBLIC_` 필수. |
| `gtag is not defined` 콘솔 에러 | Script가 안 로드됨 | 광고 차단기(AdBlock·uBlock) 끄고 시크릿 모드에서 재확인. |
| 이벤트는 잡히는데 페이지뷰가 0 | SPA 라우팅 시 `page_view` 미발생 | `lib/analytics/ga4.js` 의 `pageview()` 호출 확인. |
| 운영 환경에서만 안 잡힘 | Vercel 환경변수 미반영 | Redeploy 한 번 강제 실행. |

## 4부 — 자동 수집되는 이벤트 목록

기브니즈 사이트가 GA4로 보내는 이벤트와 각 이벤트의 의미입니다.

| 이벤트 이름 | 발생 시점 | 주요 파라미터 |
|------------|-----------|---------------|
| `page_view` | 모든 페이지 이동 (SPA 포함) | `page_path`, `page_title` |
| `lead_form_view` | 리드 폼 노출 (Intersection 기반) | `form_mode` (kakao/basic), `campaign_id` |
| `kakao_login_start` | 카카오 로그인 버튼 클릭 | `campaign_id`, `page_path` |
| `lead_form_submit` | 리드 폼 제출 성공 | `form_mode`, `campaign_id`, `lead_type` |
| `cta_click` | 일반 CTA 버튼 클릭 | `cta_label`, `cta_location`, `page_path` |

### 4.1 변환(Conversion) 등록 추천

GA4에서는 의미 있는 이벤트를 **변환**으로 표시할 수 있습니다. 다음 두 개를 추천:

1. GA4 → 관리 → 이벤트.
2. `lead_form_submit` 의 "변환으로 표시" 토글 ON.
3. `kakao_login_start` 의 "변환으로 표시" 토글 ON.

이렇게 두면 GA4 → 보고서 → 라이프사이클 → 참여 → **전환** 메뉴에서 일별 전환 수를 추적할 수 있습니다.

### 4.2 캠페인별 성과 보기

리드 폼 이벤트에 `campaign_id`가 함께 전송되므로, GA4의 **탐색** 메뉴에서:

1. 탐색 → 빈 탐색 만들기.
2. 측정기준에 `campaign_id` 추가 (맞춤 측정기준으로 사전 등록 필요 — GA4 → 관리 → 맞춤 정의).
3. 측정항목에 `이벤트 수` 추가.
4. 캠페인별로 어떤 폼이 더 잘 먹는지 한눈에 확인.

> 💡 **맞춤 측정기준 등록 방법**: GA4 → 관리 → 맞춤 정의 → 맞춤 측정기준 → 새 만들기. 이벤트 매개변수에 `campaign_id` 입력, 범위는 **이벤트**, 측정기준 이름은 `Campaign ID`.

## 부록 — 자체 추적(`tracker.js`) 와의 관계

기브니즈는 GA4 외에 **자체 추적 시스템**(`lib/tracker.js`)도 사용합니다. 둘은 보완 관계입니다.

| 항목 | GA4 | 자체 `tracker.js` |
|------|-----|-------------------|
| 데이터 보관 위치 | 구글 서버 | 우리 Supabase |
| 보관 기간 | 14개월 (기본) | 영구 |
| 어드민 연동 | 별도 GA4 UI | `/admin/leads`에서 리드와 직접 매핑 |
| 비용 | 무료 (월 1000만 이벤트까지) | 무료 (Supabase 한도 내) |
| 강점 | 채널·전환·코호트 분석 | 익명 세션 → 리드 ID 스티칭 |

**언제 어느 것을 보나**
- "이번 주 어떤 채널이 가장 잘 전환됐나" → **GA4**
- "이 특정 리드가 어디서 처음 들어와서 어디를 거쳤나" → **자체 tracker** (`/admin/leads` 행 펼침)
- "캠페인 페이지 A vs B 효과 비교" → 둘 다 (GA4: 트래픽·CTR / 자체: 실제 리드 수)

두 시스템은 서로 독립적이라 한쪽이 끊겨도 다른 쪽은 계속 돕니다.
