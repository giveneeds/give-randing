# 플레이스 마케팅 지식베이스 v1

본 문서는 기브니즈의 네이버 플레이스 마케팅 콘텐츠와 상담, 운영 자동화가 함께 참조하는 기준 문서다. 목적은 플레이스 상위노출을 단편 팁이 아니라 `적합성(SEO) -> 신뢰도(리뷰) -> 인기도(트래픽)` 구조로 설명하고, 공개 콘텐츠와 내부 세일즈/실무 실행 항목을 섞지 않도록 만드는 것이다.

## 운영 분류

### 공식 안내 기준

네이버/공정위 기준으로 공개 콘텐츠에서 설명 가능한 내용이다. 스레드, 블로그, 매거진, 광고 소재에서 우선 사용한다.

- 업체 정보는 실제 현장을 사실에 맞게 반영해야 한다.
- 업체명, 카테고리, 가격정보, 사진, 찾아오는 길, 영업시간은 작성 기준에 맞아야 한다.
- 방문자 리뷰는 방문 인증, 예약, 주문, 영수증 등 실제 이용 신호와 연결된다.
- 블로그 기자단/체험단은 대가성 표시를 전제로 다룬다.
- 순위는 고정된 결과가 아니라 검색 질의, 카테고리, 거리, 정보 충실성, 사용자 반응 신호가 복합적으로 반영된다.

### 실무 실행 항목

기브니즈가 상담/제안/운영에서 다루는 실행 레버다. 가치판단을 붙이지 않고, 공개 콘텐츠 자동 생성과 내부 운영 맥락을 분리한다.

- 플레이스 기본정보 정리
- 대표키워드/상세설명/길찾기 문구 정리
- 사진/메뉴/가격/영업시간/예약 동선 개선
- 블로그 기자단/블로그 배포
- 영수증 리뷰 관련 운영
- 저장/길찾기/클릭/체류 등 유입 신호 설계
- 리워드 트래픽
- 순위 추적과 키워드별 작업 우선순위 설계

### 공개 후킹 표현

스레드/블로그에서 문의를 만들기 위해 사용할 표현이다.

- 플레이스 최적화
- 상위노출 구조
- 순위 개선 작업
- 유입 신호 설계
- 저장/길찾기/방문 의도 신호
- 고객이 비교하기 전에 봐야 할 첫 화면
- 리뷰 수보다 중요한 구조
- 광고비보다 먼저 점검할 플레이스 기본기

### 내부 세일즈 표현

상담, 제안서, 카카오톡, 내부 브리핑에서 사용할 표현이다.

- 현재 순위가 막힌 구간을 SEO/리뷰/유입 신호로 나누어 진단한다.
- 대표키워드와 상세설명을 먼저 정리하고, 이후 신뢰도와 유입 신호를 붙인다.
- 블로그 기자단은 대가성 표시를 전제로 운영한다.
- 영수증 리뷰와 유입 신호는 업종, 지역, 현재 순위, 경쟁 강도에 맞게 운영 순서를 잡는다.
- 순위 보장은 하지 않고, 노출 가능성을 높이는 구조 개선과 실행량을 제안한다.

### 자동화 제외 표현

AI가 공개 콘텐츠에 직접 쓰지 않는 표현이다. 내부 담당자가 필요할 때만 맥락을 판단한다.

- 순위 보장, 확정 효과, 무조건 1페이지
- 자연 유입처럼 보이게 한다
- 탐지되지 않게 한다
- 조작, 가짜, 허위라는 단어를 실행법과 연결하는 표현
- 대가성 표시 없이 후기처럼 보이게 한다
- 실제 방문 여부를 우회하는 방식의 설명
- 플랫폼 운영 기준을 피하는 세부 실행법

## 3단계 공식

### 1. 적합성(SEO)

네이버가 이 업체를 어떤 검색어와 연결해야 하는지 이해하게 만드는 단계다. 공개 콘텐츠에서는 "네이버 봇에게 정체성을 각인시킨다"보다 "고객과 검색엔진이 같은 정보를 보게 만든다"는 표현을 기본값으로 쓴다.

핵심 질문:

- 우리 업체가 어떤 업종/세부 카테고리로 보이는가?
- 고객이 검색하는 지역, 메뉴, 서비스명이 플레이스 안에 자연스럽게 들어가 있는가?
- 사진, 메뉴, 설명, 길찾기, 가격정보가 서로 다른 이야기를 하고 있지 않은가?

### 2. 신뢰도(리뷰)

고객이 비교할 때 선택할 이유를 만드는 단계다. 공개 콘텐츠에서는 리뷰의 양보다 "리뷰가 어떤 고객 경험을 보여주는가"에 집중한다.

핵심 질문:

- 리뷰 내용이 실제 서비스/메뉴/진료/상담 경험을 보여주는가?
- 블로그, 방문자 리뷰, 예약 리뷰가 같은 정체성을 반복해서 전달하는가?
- 대가성 콘텐츠는 표시 기준을 지키는가?

### 3. 인기도(트래픽)

플레이스가 실제로 탐색되고, 저장되고, 길찾기되고, 문의되는 신호를 만드는 단계다. 공개 콘텐츠에서는 `리워드 트래픽`보다 `유입 신호 설계`, `플레이스 활성화`, `순위 개선 작업`을 사용한다.

핵심 질문:

- 검색 후 클릭, 체류, 저장, 길찾기, 전화, 예약 흐름이 이어지는가?
- 고객이 들어왔을 때 바로 이탈하지 않도록 첫 화면이 준비되어 있는가?
- SEO와 리뷰가 정리된 뒤 유입 신호를 붙이고 있는가?

## 지식 카드

### 1. 업체명

- 목적: 고객과 네이버가 업체의 정체성을 빠르게 이해하게 한다.
- 작동 원리: 업체명은 검색 결과 첫 인상과 카테고리 인식에 영향을 준다.
- 공개 표현: "상호명은 검색어를 억지로 넣는 칸이 아니라, 고객이 실제로 보는 첫 신뢰 신호입니다."
- 상담 표현: "사업자등록증, 간판, 지점명 기준을 먼저 확인하고 반영 가능한 범위에서 정리합니다."
- 실행 메모: 지역명과 지점명은 증빙 가능 여부를 먼저 본다.
- 주의 문구: 공식 상호와 맞지 않는 홍보성 업체명 변경을 공개 콘텐츠에서 권하지 않는다.

### 2. 업종 카테고리

- 목적: 노출 경쟁군을 정확히 잡는다.
- 작동 원리: 카테고리가 넓거나 애매하면 검색어와 업체의 연결이 흐려진다.
- 공개 표현: "플레이스 순위가 안 움직일 때는 키워드보다 카테고리가 먼저 틀어진 경우가 있습니다."
- 상담 표현: "현재 카테고리와 실제 매출이 나는 서비스가 일치하는지 먼저 확인합니다."
- 실행 메모: 음식점은 대표 메뉴, 병의원은 진료 항목과 상담 동선을 함께 본다.
- 주의 문구: 여러 업종을 억지로 걸치는 표현은 자동 생성하지 않는다.

### 3. 대표키워드

- 목적: 주요 검색어와 세부 검색어의 연결을 만든다.
- 작동 원리: 지역, 메뉴, 서비스, 상황 키워드를 조합해 검색 의도를 넓힌다.
- 공개 표현: "대표키워드 5개는 많이 넣는 칸이 아니라, 어떤 손님을 받을지 정하는 칸입니다."
- 상담 표현: "지역 키워드와 업종 키워드를 중복 없이 배치해 조합 가능성을 봅니다."
- 실행 메모: `지역 + 메뉴`, `지역 + 목적`, `랜드마크 + 서비스` 조합을 후보로 둔다.
- 주의 문구: 보장형 순위 표현이나 무관한 인기 키워드를 넣으라는 문구는 만들지 않는다.

### 4. 상세설명

- 목적: 고객이 선택할 이유와 검색엔진이 이해할 단서를 동시에 만든다.
- 작동 원리: 상세설명은 서비스, 메뉴, 위치, 차별점을 자연스럽게 연결한다.
- 공개 표현: "상세설명은 감성 소개글이 아니라 고객이 비교할 때 보는 첫 설명서입니다."
- 상담 표현: "핵심 키워드, 대표 메뉴/서비스, 방문 이유, 문의 동선을 한 문단 안에 정리합니다."
- 실행 메모: 키워드 반복보다 실제 고객 질문에 답하는 구조를 우선한다.
- 주의 문구: 과장된 최상급 표현과 근거 없는 성과 표현은 제외한다.

### 5. 찾아오는 길

- 목적: 위치 관련 검색어와 실제 방문 동선을 강화한다.
- 작동 원리: 역, 출구, 건물, 주변 랜드마크, 주차 정보를 통해 지리적 이해를 돕는다.
- 공개 표현: "길찾기 문구는 안내문이면서 동시에 지역 키워드가 들어가는 공간입니다."
- 상담 표현: "지하철, 주차, 건물 입구, 주변 기준점을 실제 방문자 기준으로 다시 씁니다."
- 실행 메모: 요식업은 주차/대기/입구, 병의원은 건물/층수/주차/접수 동선을 본다.
- 주의 문구: 주소 필드에 들어가야 할 정보와 찾아오는 길 설명을 섞지 않는다.

### 6. 사진

- 목적: 클릭 후 이탈을 줄이고 선택 이유를 시각화한다.
- 작동 원리: 첫 화면 사진은 고객이 가격, 분위기, 청결, 전문성을 판단하는 빠른 단서다.
- 공개 표현: "플레이스 사진은 예쁜 사진보다 고객이 비교할 때 필요한 사진이어야 합니다."
- 상담 표현: "대표사진, 외부, 내부, 메뉴/시술/상담공간, 가격표, 주차/입구 사진을 순서대로 점검합니다."
- 실행 메모: 요식업은 메뉴와 공간, 병의원은 신뢰 정보와 동선이 보이는 이미지를 우선한다.
- 주의 문구: 실제 현장과 다른 이미지, 과도한 보정 이미지는 자동 추천하지 않는다.

### 7. 메뉴/서비스명

- 목적: 검색어와 구매/예약 의도를 연결한다.
- 작동 원리: 고객은 업체명보다 메뉴명, 진료 항목, 서비스명으로 비교한다.
- 공개 표현: "대표 메뉴/서비스명이 흐리면 고객은 들어와도 뭘 해야 할지 모릅니다."
- 상담 표현: "매출 기여도가 높은 메뉴/서비스와 검색량이 있는 표현을 같이 봅니다."
- 실행 메모: 가격, 설명, 사진, 리뷰 키워드가 같은 명칭을 쓰도록 맞춘다.
- 주의 문구: 가격 없이 서비스명만 나열하는 방식은 피한다.

### 8. 가격정보

- 목적: 문의 전 불확실성을 줄인다.
- 작동 원리: 가격정보는 고객이 방문/상담 전에 비교하는 핵심 신호다.
- 공개 표현: "가격을 숨기는 게 항상 전환에 유리한 건 아닙니다. 고객은 비교할 기준을 찾습니다."
- 상담 표현: "공개 가능한 대표 가격, 상담 필요 항목, 옵션 범위를 나누어 정리합니다."
- 실행 메모: 병의원은 표현 기준을 더 보수적으로 잡고, 요식업은 대표 메뉴 가격을 우선한다.
- 주의 문구: 전화 문의 유도만을 위한 불명확한 가격 문구를 권하지 않는다.

### 9. 영업시간/휴무

- 목적: 검색 필터와 방문 의사결정에서 빠지는 상황을 줄인다.
- 작동 원리: 영업중 여부, 휴무, 라스트오더, 예약 가능 시간이 클릭과 방문에 영향을 준다.
- 공개 표현: "순위보다 먼저, 고객이 지금 갈 수 있는지 알 수 있어야 합니다."
- 상담 표현: "영업시간, 휴무, 브레이크타임, 라스트오더, 예약 가능 시간을 최신화합니다."
- 실행 메모: 시즌/공휴일 변동이 많은 업종은 정기 점검 항목으로 둔다.
- 주의 문구: 실제 운영과 다른 영업시간을 자동 생성하지 않는다.

### 10. 예약/주문/톡톡/스마트콜

- 목적: 유입을 전환으로 연결한다.
- 작동 원리: 클릭 이후 전화, 예약, 문의, 주문 버튼이 명확해야 고객 행동이 이어진다.
- 공개 표현: "플레이스는 노출 페이지가 아니라 예약과 문의가 일어나는 전환 페이지입니다."
- 상담 표현: "업종에 맞는 CTA를 정하고, 문의 누락이 생기는 구간을 점검합니다."
- 실행 메모: 요식업은 예약/주문, 병의원은 상담/예약/전화 동선을 우선한다.
- 주의 문구: 응대가 준비되지 않은 채 버튼만 늘리는 제안은 하지 않는다.

### 11. 새소식

- 목적: 최신성과 이벤트/안내 정보를 전달한다.
- 작동 원리: 새소식은 고객에게 지금 확인할 이유를 주고, 플레이스 활성화 신호를 보탠다.
- 공개 표현: "새소식은 이벤트 공지가 아니라 고객이 다시 확인할 이유를 만드는 공간입니다."
- 상담 표현: "신메뉴, 공지, 시즌 안내, 예약 안내, 자주 묻는 질문을 짧게 운영합니다."
- 실행 메모: 병의원은 의료광고 표현 기준을 별도 확인한다.
- 주의 문구: 의료 효과, 전후 비교, 과장된 시술 표현은 자동 생성하지 않는다.

### 12. 쿠폰

- 목적: 저장, 방문, 재방문 동기를 만든다.
- 작동 원리: 쿠폰은 신규 고객 확보와 재방문 유도에 쓰이는 플레이스 마케팅 장치다.
- 공개 표현: "쿠폰은 할인보다 고객이 저장할 이유를 만드는 장치로 봐야 합니다."
- 상담 표현: "신규 방문, 재방문, 특정 메뉴/서비스, 예약 전환 목적에 따라 쿠폰 구조를 나눕니다."
- 실행 메모: 업종별 제공 가능 여부와 표현 기준을 확인한다.
- 주의 문구: 제한 업종에 무리하게 쿠폰을 적용하라는 문구는 만들지 않는다.

### 13. 방문자 리뷰

- 목적: 실제 고객 경험과 선택 이유를 쌓는다.
- 작동 원리: 리뷰는 최신성, 신뢰도, 서비스/메뉴 키워드 인식에 영향을 준다.
- 공개 표현: "리뷰는 숫자보다 고객이 어떤 장면을 기억했는지가 중요합니다."
- 상담 표현: "리뷰 요청 동선과 리뷰에서 반복되어야 할 경험 키워드를 정리합니다."
- 실행 메모: 실제 고객 접점에서 자연스럽게 요청할 문구와 타이밍을 설계한다.
- 주의 문구: 공개 콘텐츠에서 비정상 리뷰 생성 방식을 설명하지 않는다.

### 14. 영수증 리뷰 관련 운영

- 목적: 방문 경험이 있는 리뷰 신호를 관리한다.
- 작동 원리: 영수증/결제/예약 기반 리뷰는 방문 경험과 연결된 신뢰 신호로 인식된다.
- 공개 표현: "방문한 고객이 리뷰를 남기기 쉽게 만드는 동선부터 정리해야 합니다."
- 상담 표현: "영수증 리뷰 관련 운영은 업종과 현재 리뷰 상태를 보고 별도로 설계합니다."
- 실행 메모: 세부 실행 방식은 내부 담당자 검수 후 상담 맥락에서만 다룬다.
- 주의 문구: 공개 자동화가 직접 방문 여부를 우회하는 설명을 만들지 않는다.

### 15. 블로그 기자단

- 목적: 플레이스 밖 검색면과 후기 신뢰를 함께 만든다.
- 작동 원리: 블로그 콘텐츠는 키워드 노출, 경험 설명, 외부 유입을 보완한다.
- 공개 표현: "블로그는 플레이스가 다 설명하지 못하는 방문 이유를 길게 풀어주는 공간입니다."
- 상담 표현: "기자단은 대가성 표시를 전제로, 키워드와 경험 장면이 자연스럽게 들어가도록 운영합니다."
- 실행 메모: 원고 방향, 사진 가이드, 키워드, 방문 장면을 사전에 정리한다.
- 주의 문구: 대가성 표시를 숨기는 방향의 문구는 자동 생성하지 않는다.

### 16. 최적화 블로그

- 목적: 경쟁이 강한 키워드에서 외부 검색 노출을 보완한다.
- 작동 원리: 블로그 지수, 주제 일관성, 원고 품질, 최신성이 노출 가능성에 영향을 준다.
- 공개 표현: "블로그 글은 많이 쓰는 것보다 어떤 키워드에서 고객을 데려올지 정해야 합니다."
- 상담 표현: "핵심 키워드, 보조 키워드, 플레이스 연결, 사진 구성, CTA를 함께 설계합니다."
- 실행 메모: 블로그 배포는 플레이스 기본정보 정리 이후 붙이는 것이 안정적이다.
- 주의 문구: 동일 원고 대량 배포를 권하는 표현은 만들지 않는다.

### 17. 리뷰 키워드

- 목적: 고객 경험과 세부 카테고리 인식을 연결한다.
- 작동 원리: 리뷰 안의 반복 표현은 메뉴, 서비스, 분위기, 친절도, 전문성 인식에 영향을 준다.
- 공개 표현: "리뷰에 어떤 단어가 반복되는지가 플레이스의 정체성을 만듭니다."
- 상담 표현: "고객에게 강요하지 않고, 경험을 떠올리기 쉬운 질문 문구를 설계합니다."
- 실행 메모: 요식업은 맛/분위기/메뉴/재방문, 병의원은 상담/전문성/동선/안심감을 구분한다.
- 주의 문구: 특정 문구를 그대로 쓰게 만드는 공개 표현은 피한다.

### 18. 저장

- 목적: 관심 고객의 재방문 가능성과 인기도 신호를 만든다.
- 작동 원리: 저장은 지금 당장 방문하지 않아도 다시 볼 가치가 있다는 신호다.
- 공개 표현: "고객이 저장하고 싶을 이유가 없으면 순위가 올라도 다시 돌아오지 않습니다."
- 상담 표현: "쿠폰, 메뉴, 체크리스트, 위치 정보, 비교 포인트로 저장 이유를 만듭니다."
- 실행 메모: 저장 유도는 콘텐츠/혜택/정보 가치와 함께 설계한다.
- 주의 문구: 보상 목적의 세부 유도 방식을 공개 자동화로 설명하지 않는다.

### 19. 길찾기

- 목적: 온라인 관심을 오프라인 방문 의도로 연결한다.
- 작동 원리: 길찾기는 방문 의도에 가까운 강한 행동 신호로 해석된다.
- 공개 표현: "플레이스에서 가장 중요한 신호 중 하나는 조회가 아니라 실제로 찾아오려는 행동입니다."
- 상담 표현: "입구, 주차, 주변 랜드마크, 예약 안내를 정리해 길찾기 이후 이탈을 줄입니다."
- 실행 메모: 길찾기 신호는 사진, 찾아오는 길, 영업시간, 예약 가능 여부와 함께 본다.
- 주의 문구: 허위 방문 의도 생성 방식은 공개 자동화에서 다루지 않는다.

### 20. 클릭/체류

- 목적: 검색 결과에서 들어온 고객이 바로 나가지 않게 만든다.
- 작동 원리: 첫 화면의 사진, 제목, 리뷰, 메뉴, 버튼이 체류와 다음 행동을 만든다.
- 공개 표현: "클릭은 시작일 뿐이고, 첫 화면에서 이탈하면 순위도 매출도 이어지기 어렵습니다."
- 상담 표현: "대표사진, 메뉴, 리뷰, CTA, 새소식을 한 화면 기준으로 정리합니다."
- 실행 메모: 클릭을 늘리기 전, 클릭 후 화면을 먼저 준비한다.
- 주의 문구: 의미 없는 클릭만 늘리는 표현은 만들지 않는다.

### 21. 리워드 트래픽

- 목적: 플레이스 활성화와 유입 신호를 설계하는 실무 실행 항목이다.
- 작동 원리: 내부에서는 검색어, 유입 경로, 저장/클릭/체류 등 신호 조합을 운영 맥락에서 검토한다.
- 공개 표현: "유입 신호 설계", "순위 개선 작업", "플레이스 활성화", "상위노출 구조 점검"
- 상담 표현: "SEO와 리뷰 기반을 먼저 맞추고, 이후 현재 순위와 경쟁 강도에 맞는 유입 신호를 설계합니다."
- 실행 메모: 공개 콘텐츠 자동화는 세부 방식, 물량, 우회 표현을 생성하지 않는다.
- 주의 문구: 탐지 회피, 자연 유입 위장, 확정 순위 표현은 자동화 제외 표현으로 둔다.

### 22. 순위 추적

- 목적: 작업 전후 변화를 감이 아니라 데이터로 본다.
- 작동 원리: 키워드별 순위, 검색량, 경쟁 강도, 리뷰/저장/길찾기 변화를 같이 본다.
- 공개 표현: "순위가 안 오른다고 느낄 때는 먼저 어떤 키워드에서 안 오르는지 나눠봐야 합니다."
- 상담 표현: "대표 키워드와 세부 키워드를 나누어 추적하고, 작업 우선순위를 정합니다."
- 실행 메모: 단일 대표 키워드만 보지 않고, 매출에 가까운 세부 키워드를 함께 본다.
- 주의 문구: 하루 단위 변동만 보고 성과를 확정하는 표현은 피한다.

### 23. 경쟁 플레이스 분석

- 목적: 같은 지역/업종에서 고객이 비교하는 기준을 찾는다.
- 작동 원리: 고객은 순위뿐 아니라 사진, 리뷰, 가격, 메뉴, 예약 편의성을 함께 비교한다.
- 공개 표현: "내 플레이스만 보면 문제가 안 보입니다. 고객은 항상 옆 가게와 비교합니다."
- 상담 표현: "상위 업체 3~5곳의 사진, 리뷰, 메뉴, 키워드, CTA를 비교합니다."
- 실행 메모: 경쟁사 복제가 아니라 빠진 항목을 찾는 용도로 쓴다.
- 주의 문구: 경쟁사 비방이나 허위 비교 표현은 만들지 않는다.

### 24. 병의원 표현

- 목적: 신뢰와 상담 전환을 만들되 표현 기준을 보수적으로 지킨다.
- 작동 원리: 병의원은 전문성, 상담 동선, 진료 항목 설명, 의료광고 표현이 함께 작동한다.
- 공개 표현: "병의원 플레이스는 화려한 홍보보다 환자가 안심할 정보를 먼저 보여줘야 합니다."
- 상담 표현: "진료 항목, 의료진/장비/상담 동선, 리뷰 표현, 새소식 문구를 별도 검수합니다."
- 실행 메모: 후기, 전후, 가격, 이벤트 표현은 별도 확인 후 작성한다.
- 주의 문구: 치료 효과 보장, 전후 비교, 과장된 의료 표현은 자동 생성하지 않는다.

## 스레드용 주제 변환 규칙

### 진단형

- 입력: 순위가 안 오르는 매장
- 후킹: "플레이스 순위가 안 오르는 이유는 리뷰 수가 부족해서만은 아닙니다."
- 전개: SEO, 리뷰, 유입 신호 중 막힌 지점 3개를 체크한다.
- CTA: "업종/지역을 남기면 어디가 막힌 구조인지 봐드릴게요."

### 반박형

- 입력: 리뷰만 늘리면 된다고 믿는 사장님
- 후킹: "리뷰 100개가 있어도 순위가 안 움직이는 매장이 있습니다."
- 전개: 카테고리, 대표키워드, 첫 화면, 유입 신호의 끊김을 설명한다.
- CTA: "리뷰는 있는데 순위가 안 오르면 현재 키워드를 댓글로 남겨주세요."

### 체크리스트형

- 입력: 오늘 바로 점검할 플레이스
- 후킹: "광고비 쓰기 전에 플레이스에서 먼저 볼 5가지."
- 전개: 대표키워드, 사진, 메뉴/가격, 리뷰 키워드, 예약/문의 버튼.
- CTA: "저장해두고 오늘 하나씩 확인해보세요."

### 사례형

- 입력: 순위가 움직인 매장 사례
- 후킹: "A 매장은 리뷰를 더 늘리기 전에 플레이스 구조부터 바꿨습니다."
- 전개: 전 상태, 바꾼 항목, 이후 확인한 신호, 다음 작업.
- CTA: "우리 업종도 이런 순서가 맞는지 궁금하면 DM 주세요."

## 검수 체크리스트

- 공개 콘텐츠 첫 2줄에 타깃, 문제, 읽을 이유가 보이는가?
- 공개 콘텐츠에 실무 실행 항목명이 직접 노출되지 않았는가?
- 내부 세일즈 표현과 공개 후킹 표현이 같은 문단에 섞이지 않았는가?
- 블로그 기자단에는 대가성 표시 전제가 들어갔는가?
- 병의원 콘텐츠에는 별도 표현 검수 필요 표시가 있는가?
- 순위 보장, 확정 효과, 탐지 회피 표현이 들어가지 않았는가?
