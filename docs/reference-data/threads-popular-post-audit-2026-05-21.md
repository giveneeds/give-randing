# Threads 인기 게시글 패턴 감사 2026-05-21

본 문서는 기브니즈 Threads 콘텐츠 생성 로직을 보정하기 위한 공개 검색 기반 관찰 기록이다. 이번 감사의 목적은 기존에 가정했던 `1,2,3 단계형` 글이 실제 추천/검색 결과에서 얼마나 보이는지 확인하고, 마케팅이 필요한 작은 사업자에게 맞는 글 형식과 주제 탐색어를 다시 잡는 것이다.

## 확인 범위

확인 방식:
- Threads 공개 검색 결과 카드 기준
- 로그인 없이 볼 수 있는 범위만 확인
- 게시글 상세 댓글은 미확인

검색어:
- `마케팅`
- `AI 마케팅`
- `블로그 마케팅`
- `콘텐츠 마케팅`
- `광고비`
- `자영업 마케팅`
- `네이버 플레이스`
- `인스타 광고`
- `스마트스토어`
- 결과 없음 또는 제한: `카페 바이럴`, `플레이스 마케팅`, `사장님 마케팅`, `스마트스토어 마케팅`, `홍보 어떻게`, `손님 안 와요`, `문의 안 와요`, `플레이스 순위`, `광고 대행`, `온라인 판매`

주의:
- 공개 검색 결과의 숫자는 Threads UI에 보이는 반응 수를 그대로 기록했다.
- 각 숫자의 의미는 UI 아이콘 없이 텍스트 추출한 것이므로 `visible_metrics`로만 취급한다.
- 검색 결과가 로그인 벽에서 끊기므로 계정/추천 피드 기반 감사가 별도로 필요하다.

## 발견한 패턴

### 1. 실제 검색 결과에는 `1,2,3 단계형`이 많지 않다

관찰된 글의 다수는 아래 형식이었다.

- 고백형: "마케팅 너무 어려워", "예산 0원인데 어떻게 하지"
- 질문형: "광고대행 잘 아는 사람?", "스마트스토어 상세페이지 얼마?"
- 관점전환형: "마케팅은 방법론보다 심리학"
- 금지/역후킹형: "사업하는 사람들은 보지 마세요"
- 리소스 저장형: "필수 사이트 5가지", "AI 툴 모음"
- 반박형: "네이버 망한 거 아니다"
- 비교형: "앱 광고하기 vs 광고관리자"

따라서 기브니즈 원고도 무조건 단계형으로 만들면 Threads 피드 감각과 어긋날 수 있다.

### 2. 페인포인트는 필요하지만, 꼭 교과서처럼 쓰면 안 된다

잘 보인 페인포인트는 아래처럼 날것에 가까웠다.

- "마케팅 너무 어려운거 같아"
- "홍보는 대체 어떻게 하는 거냐"
- "대행사 전화 계속 오는데 효과 있는 건가"
- "플레이스 순위가 갑자기 하락하는데 왜 이래"
- "스마트스토어 물건 올리면 팔릴 줄 알았는데 현실은 아니야"

기브니즈 적용:
- `reader_problem`은 "플레이스 최적화에 어려움이 있다"가 아니라 "플레이스 순위가 갑자기 떨어졌는데 뭐부터 봐야 할지 모르겠다"처럼 써야 한다.

### 3. 좋아요 100개 이상만 보면 저장형/전문가형으로 치우친다

좋아요/공유가 큰 글은 리소스형이 많았다.

- 사이트 리스트
- AI 툴 리스트
- 블로그 프롬프트
- 광고 라이브러리

댓글이 많은 글은 오히려 고민/질문형이 많았다.

기브니즈 적용:
- 저장형 콘텐츠와 댓글형 콘텐츠를 분리해야 한다.
- 상담 전환은 좋아요보다 댓글/DM을 목표로 잡는 편이 맞다.

### 4. 직접 키워드보다 고민어 검색이 더 중요하다

결과가 약했던 검색어:
- `플레이스 마케팅`
- `카페 바이럴`
- `사장님 마케팅`
- `광고 대행`
- `온라인 판매`

결과가 나온 검색어:
- `네이버 플레이스`
- `광고비`
- `스마트스토어`
- `자영업 마케팅`
- `인스타 광고`
- `AI 마케팅`

운영 가설:
- 사장님/운영자는 "카페 바이럴"보다 "광고비", "상세페이지", "순위 하락", "홍보 어떻게" 같은 문제어로 말한다.
- 다만 공개 검색은 문장형 고민어가 잘 안 잡히므로 계정/커뮤니티/추천 피드 기반 탐색이 필요하다.

### 5. 온라인 사장님 타겟은 확장 가능하다

스마트스토어 검색 결과에서 아래 니즈가 확인됐다.

- 상세페이지 제작
- AI에게 맡겼는데 결과물이 별로임
- 물건 올리면 팔릴 줄 알았는데 안 팔림
- 첫 주문/초보 셀러 불안

기브니즈 타겟을 `오프라인 매장`에만 고정하기보다 상위 타겟을 `고객 유입이 필요한 작은 사업자`로 두고, 하위 맥락을 나누는 것이 좋다.

- 오프라인 매장: 플레이스, 리뷰, 길찾기, 예약, 전화
- 온라인 사장님: 블로그 SEO, 카페 바이럴, 상세페이지, 콘텐츠, 광고
- 서비스/전문직: 신뢰 콘텐츠, 상담 전환, 후기, 사례

## 실행감 분류 업데이트

기존의 `실제 클릭/수정/확인 단계`만으로는 Threads 감각이 좁아질 수 있다. 관찰 결과 실행감은 4종류로 분리하는 것이 낫다.

1. 즉시 조작형
   - 예: 대표사진 바꾸기, 광고관리자 설정 확인, 상세페이지 항목 채우기

2. 자기진단형
   - 예: 내 플레이스가 어떤 키워드로 인식되는지 확인, 리뷰에 반복되는 단어 보기

3. 관점전환형
   - 예: 광고비 문제가 아니라 도착 페이지 문제일 수 있다고 이해

4. 자료저장형
   - 예: 광고 필수 사이트, AI 툴, 블로그 프롬프트, 체크리스트

## 기브니즈 생성 로직 반영안

### 브리프 생성 시

- `reader_problem`은 실제 Threads에서 보이는 날것의 말투로 만든다.
- `practical_takeaway`는 `즉시 조작형`, `자기진단형`, `관점전환형`, `자료저장형` 중 하나로 분류한다.
- `execution_steps`는 모든 글에서 UI 클릭 단계로 강제하지 않는다.
- 글 목표를 `comment`, `save`, `share`, `dm` 중 하나로 먼저 정한다.

### 스레드 초안 생성 시

첫 줄은 아래 중 하나로 시작하는 것을 우선한다.

- 고백 대리형: "마케팅 너무 어려운 사장님들,"
- 문제 직격형: "플레이스 순위가 갑자기 떨어졌다면,"
- 반박형: "네이버 이제 끝났다는 말, 아직 이릅니다."
- 금지 후킹형: "광고비 쓰기 전에 이건 보지 마세요."
- 비교형: "인스타 앱 광고와 광고관리자, 차이 모르면 돈 샙니다."
- 자료 약속형: "사장님이 저장해둘 마케팅 사이트 5개."

### CTA 선택

- 댓글형: 페인포인트/질문형 글에 사용
- 저장형: 체크리스트/리소스형 글에 사용
- DM형: 비교형/진단형 글에 사용
- 공유형: 동료 사장님에게 보내기 좋은 기준 글에 사용
