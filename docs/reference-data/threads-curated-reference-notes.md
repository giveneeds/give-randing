# Threads Curated Reference Notes

이 파일은 잘 쓴 Threads 발행물을 DB 분석값으로 저장하기 전 단계의 큐레이션 노트다.

목적은 LLM 이 원고를 만들 때 "이런 식으로 구성되는구나"를 읽게 하는 것이다. 문장을 베끼지 말고 구성 감각만 빌린다.

## 사용 원칙

- 좋은 스레드 원문을 발견하면 긴 분석보다 짧은 관찰 카드로 추가한다.
- 첫 포스트가 왜 멈춰 세우는지 먼저 적는다.
- 2번 이후 포스트가 어떤 역할로 이어지는지 적는다.
- 저장/공유/댓글이 생길 이유를 적는다.
- 우리 콘텐츠 기둥에 어떻게 옮길 수 있는지 적는다.
- DB처럼 정규화하려고 오래 쓰지 않는다. 프롬프트에 들어갈 수 있게 짧고 선명해야 한다.

## 관찰 카드 형식

```text
### REF-YYYYMMDD-이름
- source_url:
- author:
- topic:
- 왜 볼 만한가:
- 첫 포스트 구성:
- 후속 포스트 구성:
- 반응 장치:
- 우리 기둥 매핑:
- 우리 글에 빌릴 점:
- 피할 점:
```

## REF-20260526-auto-gongbang-crm

- source_url: docs/reference-data/threads-reference-detail-2026-05-26-auto-gongbang-candidates.json
- author: auto.gongbang
- topic: 운영 CRM 제작 과정 공유
- 왜 볼 만한가: "작업하며 정리한 관점"으로 시작해서 실제 만든 시스템의 구성 요소를 1/n로 풀어낸다. 독자는 결과물이 아니라 만드는 과정을 따라 보게 된다.
- 첫 포스트 구성: 무엇을 만들고 있는지, 누구를 위해 만드는지, 어떤 관점을 공유할지 짧게 연다.
- 후속 포스트 구성: 자동화, 데이터 연동, 매일 쓰는 동선, Google 환경, 테스트 모드처럼 각 포스트가 기능 하나와 운영상 의미 하나를 맡는다.
- 반응 장치: "나도 저런 시스템 있으면 편하겠다"는 저장 욕구. 기능 설명이지만 실제 운영 장면이 있어 추상적이지 않다.
- 우리 기둥 매핑: do_today, content_showcase
- 우리 글에 빌릴 점: 도구/AI/마케팅 팁을 말할 때 기능 목록으로 끝내지 말고 "운영자가 평소처럼 일하고, 도구가 따라오는 장면"을 보여준다.
- 피할 점: 우리 글에서 실제 구현 사례가 없는데 있는 것처럼 꾸미면 안 된다.

## REF-20260526-gptaku-fomo

- source_url: 사용자 제공 Threads 예시, docs/reference-data/threads-reference-detail-2026-05-26-gptaku-candidates.json
- author: gptaku_ai
- topic: FOMO를 일으키는 방식
- 왜 볼 만한가: 한 개념을 여러 분야로 반복 적용해 독자가 패턴을 이해하게 만든다.
- 첫 포스트 구성: "FOMO를 일으키는 방법 8가지"처럼 저장할 이유를 바로 준다.
- 후속 포스트 구성: AI, 주식, 부동산, 교육, 커리어, 자기계발, 소비재처럼 분야만 바꾸고 같은 구조를 반복한다. 각 포스트는 "무엇을 보여준다 → 어떻게 말한다 → 어떤 감정을 만든다"로 닫힌다.
- 반응 장치: 독자가 다른 사례를 떠올리게 만드는 패턴화. 정보라기보다 세상을 보는 프레임을 준다.
- 우리 기둥 매핑: current_observation, trend_plain
- 우리 글에 빌릴 점: "A를 분석하게 만드는 게 아니라 B라는 감정을 만든다"처럼 표면 행동과 실제 감정을 분리한다.
- 피할 점: 같은 문장 구조를 모든 글에 복붙하면 금방 작위적으로 보인다.

## REF-20260526-choi-openai

- source_url: docs/reference-data/threads-detail-manual-choi-openai-DYuVvqtj8Av.json
- author: choi.openai
- topic: AI/자동화/업무 방식 관련 정보형 스레드
- 왜 볼 만한가: 독자가 모를 수 있는 개념을 처음부터 전문 용어로 밀지 않고, 흐름을 따라 이해하게 만든다.
- 첫 포스트 구성: 새로 보이는 현상이나 도구를 바로 설명하지 않고, "왜 이걸 봐야 하는지"를 먼저 만든다.
- 후속 포스트 구성: 개념 설명, 예시, 실제 쓰임, 주의점, 기억할 기준으로 나뉜다.
- 반응 장치: 새로운 개념을 놓치고 싶지 않은 감각과 저장 욕구.
- 우리 기둥 매핑: trend_plain, do_today
- 우리 글에 빌릴 점: AI/마케팅 도구 글은 "무엇이다"보다 "그래서 일하는 방식이 어떻게 달라지는가"로 풀어야 한다.
- 피할 점: 확인되지 않은 기능이나 성과를 만들어내지 않는다.

## 새 레퍼런스 추가 규칙

새 예시는 이 파일 맨 아래에 추가한다. 좋은 원문 전체는 `docs/reference-data`의 json/md에 두고, 이 파일에는 생성에 바로 도움이 되는 관찰만 남긴다.

- 우리 글에 빌릴 점 1~2줄
