# Research Layers

스레드 생성은 단일 검색으로 끝내지 않는다. 뉴스 기사만 모아오면 소재는 빨라지지만 해외에서 먼저 움직이는 마케팅/AI 이슈, 도구 활용, 실험 사례를 놓치기 쉽다.

## 1차 수집: 주제 후보 풀

목적은 오늘 발행할 만한 주제를 넓게 모으는 것이다. 이 단계에서 Reddit/X를 보는 이유는 말투를 보기 위해서가 아니라, 해외 마케팅/AI 커뮤니티와 실무자들이 이미 올려둔 주제, 논쟁, 도구 활용, 이슈를 발견하기 위해서다.

- 기존 뉴스/웹 수집은 유지한다.
- Reddit/X 비중을 높이기 위해 1차 텔레그램 제안에는 Reddit 후보 1개, X 후보 1개를 추가 시도한다.
- Reddit/X 후보는 "해외 주제 후보"다. 해외에서 이런 식으로 쓰고 있다, 이런 도구/이슈가 올라온다, 한국어로 보기 좋게 정리하면 콘텐츠가 될 수 있다는 관점으로 평가한다.
- Reddit/X 후보를 별도 섹션으로 격리하지 않는다. 일반 후보 흐름 안에서 출처와 활용 가능성을 밝힌다.
- 점수가 낮아도 삭제하지 않는다. 한국어로 재해석할 수 있는 소재인지, 기브니즈 관점으로 풀 수 있는지 기록한다.
- 이 단계에서는 한국 Threads 말투를 판단하지 않는다.

## 1.5차 주제 발굴 트리거

Reddit/X 후보를 만들 때는 아래 트리거 중 하나를 사용한다.

- `ai_marketing`: AI 마케팅, 자동화, 에이전트, 프롬프트/워크플로우 활용
- `creator_tools`: 콘텐츠 제작 도구, 숏폼/스레드/이미지/영상 제작 자동화
- `growth_case`: 스타트업/소상공인/브랜드 성장 실험
- `platform_shift`: 검색, SNS, 광고, 커머스 플랫폼 변화
- `campaign_issue`: 해외 브랜드 캠페인, 논란, 소비자 반응, 밈화
- `operator_pain`: 마케터/운영자가 겪는 비용, 시간, 품질, 반복 작업 문제

각 트리거는 하위 리서치 에이전트처럼 작동한다. 입력은 기존 수집 후보와 활성 주제의 키워드이고, 출력은 `agent_items`에 저장 가능한 후보 1개 이상이다.

## 2차 리서치: 선택 주제의 보강 맥락

사용자가 주제 후보를 고른 뒤 실행한다.

- `runDeepResearch`가 Reddit, X, Threads, 웹을 우선 포함해 해외/소셜 맥락과 후킹 포인트를 본다.
- 통합 검색과 별개로 Reddit 전용, X 전용 검색을 추가한다.
- 목표는 "해외에서 이 주제가 어떤 식으로 다뤄지는가", "한국어 콘텐츠로 가져오면 어떤 앵글이 자연스러운가", "반응/반박/궁금증이 있다면 무엇인가"를 찾는 것이다.

## 2.5차 리서치: 추가 자료 보강

2차 리서치가 주제의 확산 맥락과 앵글을 본다면, 2.5차는 본문에 들어갈 근거와 사례를 보강한다.

- `runSupplementalResearch`가 웹 전체에서 최신 근거, 사례, 빠진 맥락을 찾는다.
- `includeDomains: null`을 써서 기본 SNS 도메인 제한을 풀 수 있다.
- `PERPLEXITY_API_KEY`가 있으면 Perplexity Sonar API로 추가 근거를 받는다.
- Perplexity가 없거나 실패해도 Tavily 검색 결과만으로 계속 진행한다.
- 결과는 `evidence_points`, `content_additions`, `missing_context`로 정리한다.

## 3차 리서치: 말투/VOC

후보 생성 직전에 실행한다. 다만 이 단계는 매번 필수 라이브 검색일 필요는 없다. 주차별로 Threads 원고/인기글을 50개씩 수집해 둔 감사 데이터가 있으면 그 데이터를 우선 사용하고, 부족할 때만 추가 검색한다.

- `runToneResearch` 또는 기존 Threads 감사 데이터가 한국 Threads의 후킹 포인트, 문장 길이, 시작 방식, 피해야 할 AI식 표현을 본다.
- 목표는 문장을 베끼는 것이 아니라 한국어 Threads에 맞는 리듬을 잡는 것이다.
- Reddit/X는 3차 말투 리서치의 중심 소스가 아니다. 한국 뉘앙스는 Threads 감사 데이터와 국내 반응을 우선한다.
- 결과는 `voice_patterns`, `phrases_to_borrow`, `phrases_to_avoid`, `reader_questions`로 정리한다.

## 에이전트 역할

- `Topic Scout`: 뉴스/웹/Reddit/X에서 후보 주제를 찾는다.
- `Overseas Signal Scout`: Reddit/X에서 해외 마케팅/AI 이슈와 도구 활용 사례를 찾는다.
- `Planner`: 주제 후보를 점수화하고 사용자에게 선택지를 보낸다.
- `Deep Researcher`: 선택 주제의 해외/소셜 맥락과 후킹 포인트를 보강한다.
- `Evidence Researcher`: 추가 근거, 사례, 빠진 맥락을 찾는다.
- `Tone Adapter`: 한국 Threads 말투와 후킹 감각을 맞춘다. 기존 Threads 감사 데이터가 있으면 우선 사용한다.
- `Variant Generator`: 기둥 2~3개 안에서 겹치지 않는 7개 후보를 만든다.
- `Quality Reviewer`: 7개 후보를 검수하고 부족한 후보를 보강한다.
- `Orchestrator`: 별도 LLM 에이전트라기보다 현재는 `finishPlanningSession` 같은 코드 오케스트레이터가 담당한다. 상태 잠금, 순서 실행, 저장, 텔레그램 보고를 맡는다.

## 후보 생성 후 품질 검수

7개 후보가 만들어진 뒤 한 번 더 검수한다.

- 후보 7개가 서로 겹치면 실패다.
- 기둥 2~3개가 실제로 분산됐는지 본다.
- FOMO가 과격하거나 억지로 붙었는지 본다.
- `single_post`가 500자 미만이면 실패다.
- Q/A 형식, 일반론, 억지 자영업자 팁, 출처 없는 단정은 고친다.
- 검수 결과는 `variant_review.metadata.quality_review`에 남긴다.

## 저장 원칙

- 점수가 낮은 후보도 삭제하지 않는다.
- 추천 후보는 `recommended_variant_id`로 표시할 뿐 최종 선택이 아니다.
- 사용자가 7개를 보고 하나 또는 여러 개를 선택해야 한다.
