# Format Length And Conversational Explainer

글의 길이는 기둥으로 결정하지 않는다. 소재의 밀도, 독자의 기대, 말하고 싶은 한 문장의 힘으로 결정한다.

## Format Type

### `single_post`

적합한 주제:
- 공감형 문제 제기
- 짧은 오해 깨기
- 한 장면과 한 관점으로 충분한 상황
- 댓글 유도 질문

분량:
- 최소 500자 이상.
- 권장 500~900자.
- 한 포스트 안에 장면, 맥락, 관점, 마무리가 모두 있어야 한다.

성공 신호:
- 댓글에서 자신의 상황을 말한다.
- 저장보다 답글/공감이 많다.

### `short_thread`

적합한 주제:
- 독자가 이해하려면 2~7개의 장면, 원인, 사례, 판단 기준이 필요하다.
- 단일글로 쓰면 정보가 뭉개진다.
- 각 포스트가 독립된 역할을 가진다.

### `resource_thread`

적합한 주제:
- 도구, 프롬프트, 예시, 템플릿, 검색어처럼 모을수록 가치가 커진다.
- 목록 자체가 저장 자산이다.
- 리드마그넷 없이도 저장 가치가 있다.

## 포스트 수 결정 게이트

- 1개: 한 포스트 안에 장면, 맥락, 관점, 마무리를 500자 이상으로 충분히 담을 수 있을 때.
- 2~3개: 단순히 "기준 정도면 충분"이 아니라, 포스트마다 서로 다른 역할이 있을 때만 쓴다. 예: 문제 제기 / 원인 설명 / 판단 기준.
- 4개: 서로 다른 사례나 판단 기준이 3개 이상 있고, 각 글이 저장 가능한 미니 카드일 때.
- 5~7개: 새 개념/뉴스/기술 또는 논쟁적 소재를 독자가 처음 이해해야 하고, 배경, 쉬운 설명, 사례, 의미, 판단 기준이 필요할 때.
- 8개 이상: 도구/문장/검색어/템플릿/업종별 예시처럼 목록 자체가 자산이거나, 사례가 많을 때만 허용한다.

## 길이 결정 게이트

- 짧게 쓰는 기준: 독자가 이미 감정을 알고 있고, 새 관점 하나만 얹으면 될 때.
- 길게 쓰는 기준: 독자가 왜 중요한지 모를 수 있거나, 잘못된 상식/비교/예시가 있어야 납득되는 때.
- 나눠 쓰는 기준: 글을 나누지 않으면 장면, 원인, 기준, 예시 중 하나가 흐려지는 때.
- 길게 쓰면 안 되는 기준: 같은 결론을 다른 말로 반복하거나, 예시 없이 주장만 늘어나는 때.

## 대화형 설명 전개 기준

- Threads는 한 포스트에 1000자까지 쓸 수 있으므로 500자 안에 억지로 압축하지 않는다.
- 정보글은 포스트당 350~850자까지 허용한다.
- 새 개념은 문답지처럼 쓰지 않는다. 독자가 속으로 물을 법한 질문을 자연스러운 설명문으로 녹인다.
- `Q. / A.` 형식은 쓰지 않는다.
- 대화형 설명을 고른 경우 첫 2개 포스트 안에서 반드시 "그래서 이게 뭔데?"에 해당하는 쉬운 설명을 준다.
- 독자가 기사 원문을 보지 않았다고 가정한다. 약어, 제품명, 기술명, 행사명은 처음 등장할 때 쉬운 말로 번역한다.
- 대화형은 억지 캐릭터극이 아니라 독자가 속으로 물을 질문을 대신 던지는 방식이다.
- "아직 구경 중일 때가 많습니다", "진짜 오늘 나갈 사람은"처럼 일상 언어가 아닌 문장은 탈락시킨다.

## CTA

- CTA는 기본값이 아니다.
- `current_observation`, `trend_plain`은 여운형/저장형이 기본이다.
- `do_today`, `content_showcase`는 저장형이 기본이다.
- `cost_before_spend`는 질문형 또는 저장형이 기본이다.
- 상담형 CTA는 진단형 글에서만 사용한다.
- 본문과 이어지지 않는 "업종/지역 남겨주세요" 반복 CTA는 금지한다.
