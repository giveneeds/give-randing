# Threads Content Pattern Harness

본 문서는 기브니즈 스레드 콘텐츠 생성 로직을 만들기 위한 레퍼런스 관찰 기준이다. 목표는 인기 게시글을 감으로 따라 하는 것이 아니라, "어떤 주제는 어떤 글 형식이 맞는가"를 데이터로 쌓아 초안 생성 에이전트가 참조하게 만드는 것이다.

## 목표

기브니즈의 1차 채널은 Threads다. 매거진/블로그는 잠시 후순위로 두고, 오늘 단계에서는 스레드에서 반응 가능성이 있는 주제와 형식을 먼저 안정화한다.

좋은 스레드 초안의 기준:
- 독자가 "내 가게/병원에도 적용해볼 수 있겠다"고 느낀다.
- 전문가용 일반론보다 작은 사업자가 이해할 수 있는 말로 쓴다.
- 단일글인지 연결글인지 먼저 판단한다.
- 연결글은 긴 글을 쪼갠 것이 아니라, 각 게시글이 저장 가능한 미니 카드처럼 작동해야 한다.
- 리드마그넷은 꼭 필요할 때만 붙인다. 억지 체크리스트 PDF는 실패로 본다.

## 현장어 치환 원칙

스레드는 전문 용어를 설명하는 채널이 아니라, 사장님이 이미 느끼는 답답함을 말로 잡아주는 채널이다. 생성기는 아래 원칙을 모든 주제에 공통 적용한다.

- 전문 용어를 먼저 쓰지 않는다. 필요하면 쉬운 말 뒤에 괄호로만 보조한다.
- `reader_problem`은 보고서 문장이 아니라 현장에서 나오는 고민 문장으로 쓴다.
- "전환율", "퍼널", "ROAS", "세그먼트", "포지셔닝", "브랜딩", "최적화" 같은 말은 그대로 쓰기 전에 생활어로 바꾼다.
- 문장 첫 줄은 지식 전달보다 공감, 의심, 불안, 손해감, 저장 가치를 먼저 건드린다.
- 좋은 문장은 "마케팅 효율이 낮다"가 아니라 "광고비는 나가는데 전화가 안 온다"에 가깝다.

여기서 말하는 현장어는 반말이나 과한 유행어가 아니다. 아래 기준 중 2개 이상이 들어간 문장을 우선한다.

- 돈: 광고비, 대행비, 매출, 예약, 문의, 전화, 상담.
- 시간: 이번 달, 요즘, 3개월째, 매일 올리는데, 주말마다.
- 화면: 플레이스 첫 사진, 리뷰, 블로그 제목, 전화 버튼, 예약 버튼, 메뉴판, 상세페이지.
- 행동: 올렸다, 바꿨다, 맡겼다, 물어봤다, 찍었다, 답글 달았다.
- 감정: 답답하다, 불안하다, 모르겠다, 믿기 어렵다, 손해 보는 것 같다.

치환 예시:

| 전문 용어 | 스레드용 표현 |
|---|---|
| 전환율이 낮다 | 들어오긴 하는데 문의가 안 온다 |
| 퍼널이 끊긴다 | 손님이 중간에 빠져나간다 |
| ROAS가 안 나온다 | 광고비 쓴 만큼 돌아오는 게 없다 |
| 타깃 세그먼트 | 우리 가게를 고를 사람 |
| 브랜드 포지셔닝 | 손님 머릿속에 남는 한마디 |
| 콘텐츠 최적화 | 사람들이 보고 넘기지 않게 고치는 것 |
| 플레이스 SEO | 네이버에서 우리 가게가 잘 보이게 정리하는 것 |
| 고객 여정 | 손님이 검색하고 예약하기까지 보는 순서 |
| 리드 전환 | 문의나 예약으로 이어지는 것 |
| 신뢰 신호 | 여기 괜찮겠다고 느끼게 하는 단서 |

나쁜 예:

- "AI 기반 마케팅 자동화로 고객 전환율을 개선해야 합니다."
- "플레이스 최적화를 통해 유입 퍼널을 강화해야 합니다."
- "브랜드 포지셔닝을 명확히 해야 합니다."

좋은 예:

- "AI한테 광고 문구부터 시키기 전에, 손님이 왜 전화 버튼까지 안 가는지 먼저 물어보세요."
- "플레이스 사진은 많은데, 첫 사진만 보고는 뭐가 좋은 가게인지 안 보일 수 있습니다."
- "광고비는 쓰는데 예약이 안 늘면, 광고보다 먼저 손님이 보는 첫 화면을 봐야 합니다."

## 레퍼런스 수집 단위

Threads 게시글을 볼 때는 본문만 저장하지 않는다. 생성 로직에 필요한 판단 근거를 함께 기록한다.

  "topic": "AI 활용 사례 큐레이션",
  "audience_guess": "creator | small_business_owner | marketer | clinic_owner | restaurant_owner | general",
  "visible_metrics": {
    "views": 160000,

  "content_pattern": "list | checklist | case_breakdown | tool_roundup | before_after | opinion | mini_guide",

## 콘텐츠 기둥

기브니즈 Threads는 플레이스 전용 계정이 아니다. 작은 사업자가 마케팅에 돈을 쓰기 전, 직접 보고 판단하고 오늘 하나라도 고칠 수 있게 돕는 계정이다. 아래 5개 기둥을 번갈아 사용한다.

### 1. 광고비 쓰기 전에 먼저 봐야 할 것

역할:
- 마케팅 비용을 날리지 않게 돕는다.
- 대행사/광고/보고서를 무조건 불신하게 만드는 것이 아니라, 돈 쓰기 전에 사장님이 확인할 수 있는 질문을 준다.

주제 예시:
- 광고비 쓰기 전 체크.
- 마케팅 맡기기 전에 먼저 물어볼 것.
- 보고서에서 봐야 할 숫자.
- 월 50만 원 예산이면 먼저 할 일.
- 성과가 안 날 때 광고 탓만 하면 안 되는 이유.

권장 마무리:
- 저장형, 질문형, 여운형.
- 상담 CTA는 꼭 진단형 글일 때만 쓴다.

### 2. 오늘 바로 해볼 수 있는 마케팅

역할:
- 팔로우 이유를 만든다. "나중에 써먹어야겠다"는 저장 가치를 준다.

주제 예시:
- AI로 콘텐츠 소재 뽑기.
- 리뷰 답글 바꾸기.
- 블로그 제목 다시 쓰기.
- 메타 광고 소재 3개로 나눠보기.
- 상세페이지 첫 문장 고치기.
- 플레이스 첫 사진 교체하기.

권장 마무리:
- 저장형, 실행 제안형.
- "오늘 10분만 해볼 것"처럼 작게 닫는다.

### 3. 요즘 되는 방식에 대한 관찰

역할:
- 사장님 실무 팁을 넘어 일반인도 재미있게 읽을 수 있는 글을 만든다.
- "요즘 사람들은 왜 이런 걸 좋아하지?"에 가까운 현상 관찰이다.

주제 예시:
- 요즘 잘 되는 가게들은 왜 설명을 짧게 할까.
- 요즘 사람들은 광고보다 댓글을 먼저 믿는 것 같다.
- 요즘 인기 있는 계정들은 완벽해서가 아니라 자주 보여서 기억된다.
- 요즘 소비자는 싼 곳보다 덜 불안한 곳을 고른다.
- AI 시대가 되니까 오히려 사람 냄새 나는 글이 더 잘 읽힌다.
- 스레드에서 뜨는 글들은 정보보다 "내 얘긴데?"가 먼저 온다.

권장 마무리:
- 여운형, 공감 질문형.
- 억지 상담 CTA 금지. 이 기둥은 계정의 폭을 넓히는 역할이다.

### 4. 요즘 마케팅 흐름 쉽게 풀기

역할:
- 트렌드를 뉴스 요약으로 끝내지 않고, 그래서 사장님이 무엇을 보면 되는지 쉽게 바꾼다.

주제 예시:
- AI 마케팅을 실제로 쓰는 방식.
- 검색이 바뀌면 사장님이 봐야 할 것.
- 스레드/숏폼/블로그 역할 차이.
- 메타 광고 소재가 바뀌는 흐름.
- 네이버 플레이스 변화.
- 소비자가 요즘 비교하는 방식.

권장 마무리:
- 저장형, 관점 정리형.
- "이 흐름에서 사장님이 볼 것은 하나입니다"처럼 정리한다.

### 5. 우리 업체를 더 잘 보여주는 콘텐츠 만들기

역할:
- 기브니즈의 콘텐츠 제작 역량을 보여준다.
- 단순히 많이 올리는 법이 아니라, 무엇을 어떻게 보여줄지 기준을 준다.

주제 예시:
- 광고 소재 만드는 법.
- 후킹 문장 만드는 법.
- 고객 사례를 콘텐츠로 바꾸는 법.
- 사장님의 전문성을 쉽게 풀어내는 법.
- 가격보다 선택 이유를 보여주는 법.
- 블로그/스레드/카드뉴스 소재 고르는 법.

권장 마무리:
- 저장형, 예시 요청형.
- 진단 CTA를 쓰더라도 "업종 남겨주세요" 같은 반복 문구는 피하고, 본문에서 다룬 소재 기준과 이어지게 쓴다.

운영 비중 기본값:

| 기둥 | 비중 |
|---|---:|
| 광고비 쓰기 전에 먼저 봐야 할 것 | 15% |
| 오늘 바로 해볼 수 있는 마케팅 | 30% |
| 요즘 되는 방식에 대한 관찰 | 20% |
| 요즘 마케팅 흐름 쉽게 풀기 | 20% |
| 우리 업체를 더 잘 보여주는 콘텐츠 만들기 | 15% |

생성기는 글을 쓰기 전에 먼저 아래를 결정한다.

```json
{
  "content_pillar": "cost_before_spend | do_today | current_observation | trend_plain | content_showcase",
  "content_goal": "save | comment | share | viewpoint | dm",
  "ending_type": "none | save | question | soft_dm | diagnostic_comment",
  "why_this_ending": ""
}
```

CTA 규칙:
- CTA는 기본값이 아니다.
- `current_observation`, `trend_plain`은 여운형/저장형이 기본이다.
- `do_today`, `content_showcase`는 저장형이 기본이다.
- `cost_before_spend`는 질문형 또는 저장형이 기본이다.
- 상담형 CTA는 진단형 글에서만 사용한다.
- 본문과 이어지지 않는 "업종/지역 남겨주세요" 반복 CTA는 금지한다.

## 글 형식 분류

### single_post

적합한 주제:
- 공감형 문제 제기
- 짧은 오해 깨기
- 사장님이 바로 떠올릴 수 있는 상황
- 댓글 유도 질문

구조:
- 1문장 후킹
- 문제 상황
- 관점 전환
- 짧은 실행 힌트

예시 주제:
- "광고비를 올렸는데 문의가 안 늘어나는 이유"
- "플레이스 사진을 바꿔야 하는 타이밍"
- "AI로 마케팅을 시작할 때 처음 하면 좋은 일"

성공 신호:
- 댓글에서 자신의 상황을 말한다.
- 저장보다 답글/공감이 많다.

### short_thread

적합한 주제:
- 실행 단계가 3~5개로 나뉜다.
- 독자가 저장해두고 따라 할 수 있다.
- 단일글로 쓰면 정보가 뭉개진다.

구조:
- 1번: 왜 이 주제가 중요한지와 읽을 이유
- 2~4번: 단계별 실행법 또는 사례
- 마지막: 오늘 할 수 있는 작은 행동

예시 주제:
- "동네 병원이 상담 전환을 높이는 콘텐츠 4단계"
- "식당 사장님이 네이버 플레이스에서 먼저 점검할 5가지"
- "AI로 이번 달 콘텐츠 주제를 뽑는 방법"

성공 신호:
- 저장/공유가 상대적으로 높다.
- 각 연결 게시글이 독립적인 팁으로 읽힌다.

### resource_thread

적합한 주제:
- 툴, 프롬프트, 예시, 템플릿처럼 모을수록 가치가 커진다.
- 독자가 바로 열어보거나 따라 할 수 있다.
- 리드마그넷 없이도 저장 가치가 있다.

구조:
- 1번: 결과 약속과 대상 명시
- 2~6번: 리소스별 용도, 적용 상황, 주의점
- 마지막: 어떤 순서로 써볼지 안내

예시 주제:
- "작은 가게가 AI로 마케팅할 때 써볼 도구 5개"
- "병원 블로그 소재를 뽑을 때 쓰는 질문 7개"
- "플레이스 개선 전에 봐야 할 무료 점검 항목"

성공 신호:
- 저장/공유/리포스트가 높다.
- 댓글에서 자료 요청이 나온다.

### mega_thread

적합한 주제:
- 강한 큐레이션 자산이 이미 있다.
- 여러 출처와 사례를 묶어야 한다.
- 하루짜리 일반 콘텐츠보다 캠페인성 콘텐츠에 가깝다.

주의:
- 기브니즈의 매일 운영 기본값으로 두지 않는다.
- 10개 이상 연결글은 완성도와 검수 비용이 커진다.
- 초기 기본값은 3~5개 연결글이다.

## 주제별 전개 방식

### 플레이스 노출

추천 형식:
- short_thread
- resource_thread

전개:
- "노출이 안 된다"보다 "고객이 비교할 때 무엇을 보는가"로 시작한다.
- 사진, 리뷰, 메뉴/시술명, 첫 화면 문구, 예약/문의 동선으로 쪼갠다.
- 업종별로 예시는 다르게 준다.

요식업 적용:
- 메뉴 사진, 대표 메뉴명, 리뷰 답글, 재방문 쿠폰, 저장하고 싶은 정보.

병의원 적용:
- 진료 항목명, 의료진 신뢰 정보, 상담 전환 문구, 후기의 표현 리스크.

### 광고 효율

추천 형식:
- single_post
- short_thread

전개:
- 광고비 낭비 공감에서 시작한다.
- "광고 세팅"보다 "광고 이후 도착 페이지/상담/플레이스가 준비됐는가"를 본다.
- CPC보다 전환 흐름을 설명한다.

주의:
- 확정 수익, 과장된 ROAS 표현은 피한다.

### 콘텐츠 마케팅

추천 형식:
- short_thread
- resource_thread

전개:
- "무엇을 써야 할지 모르겠다"는 문제에서 시작한다.
- 사장님의 전문성/친근함/사례를 콘텐츠 소재로 바꾸는 방법을 보여준다.
- 예시 제목을 바로 준다.

### AI 마케팅 활용

추천 형식:
- resource_thread
- short_thread

전개:
- AI 도구 소개보다 "오늘 이 업무에 어떻게 쓰는가"를 먼저 말한다.
- 프롬프트, 입력값, 결과물, 수정 기준을 같이 준다.
- 요식업/병의원 모두 적용 가능하면 예시를 각각 1개씩 둔다.

주의:
- "AI로 매출 자동화" 같은 과장형 후킹은 피한다.
- 도구명 나열보다 사장님의 실제 업무에 붙인다.

### 재방문/상담 전환

추천 형식:
- single_post
- short_thread

전개:
- 신규 유입보다 놓치고 있는 전환 지점을 보여준다.
- 예약 전, 방문 직후, 리뷰 요청, 재방문 메시지처럼 고객 여정으로 쪼갠다.

## 후킹 심리 패턴

### 불안 재구성형

참조 관찰:
- 2026-05-20 수동 입력 레퍼런스: `sangsang.cci`의 5개 연결글.
- 핵심 패턴은 "사람은 결핍을 해결하려고만 소비하는 것이 아니라, 뒤처질까 봐 불안을 줄이기 위해 소비한다"는 관점이다.
- 반응 포인트는 특정 상품 소개가 아니라, 독자가 이미 느끼던 막연한 불안을 문장으로 잡아주는 데 있다.

기브니즈 적용:
- 사장님에게 "지금 안 하면 망한다"는 공포를 과장하지 않는다.
- 대신 "왜 계속 광고비를 쓰는데도 불안한지", "왜 블로그/리뷰를 해도 확신이 안 생기는지"를 진단 가능한 구조로 바꾼다.
- 불안의 원인을 개인 능력 부족이 아니라 플레이스/콘텐츠/유입 구조의 끊김으로 설명한다.
- 마지막에는 불안을 소비로 밀어붙이지 않고, 오늘 확인할 수 있는 작은 점검 항목이나 댓글/DM 진단으로 연결한다.

추천 형식:
- short_thread
- single_post

전개:
- 1번: 사장님의 막연한 불안을 정확히 언어화한다.
- 2번: 그 불안이 생기는 구조를 설명한다.
- 3번: 흔한 잘못된 대응을 짚는다.
- 4번: 진짜 봐야 할 지표/화면/동선을 제시한다.
- 5번: 오늘 할 수 있는 작은 행동이나 진단 CTA로 닫는다.

예시 후킹:
- "마케팅을 하고 있는데도 계속 불안한 이유는, 돈을 안 써서가 아니라 어디가 막혔는지 안 보이기 때문입니다."
- "플레이스 순위가 안 오르면 사장님들은 더 많은 리뷰, 더 많은 블로그, 더 많은 광고비를 떠올립니다. 그런데 진짜 문제는 양이 아니라 구조일 때가 많습니다."
- "요즘 사장님들이 가장 많이 사는 건 광고가 아니라 안심입니다. 문제는 그 안심이 실제 고객 유입으로 이어지는 구조인지 확인하지 않는다는 겁니다."

주의:
- 과도한 위기감, 사회 전체 공포, 확정적 몰락 표현은 쓰지 않는다.
- 불안을 자극한 뒤 바로 구매를 압박하지 않는다.
- 불안을 "점검 가능한 문제"로 낮춰주고, 실행 기준을 함께 제시한다.

## 초안 생성 로직에 반영할 규칙

1. 먼저 채널을 판단한다.
   - 오늘은 기본값을 `threads`로 둔다.
   - 매거진 초안 생성은 승인 이후 별도 단계로 미룬다.

2. 다음으로 형식을 판단한다.
   - 실행 팁이 1개면 `single_post`.
   - 실행 팁이 3개 이상이면 `short_thread`.
   - 툴/자료/예시가 5개 이상이면 `resource_thread`.
   - 강한 큐레이션 자산이 없으면 `mega_thread`를 선택하지 않는다.

3. 텔레그램 검수 카드에는 형식 판단 이유를 노출한다.
   - "이 소재는 스레드 5개 연결글이 맞습니다."
   - "저장 가치가 있는 실행 단계가 4개로 나뉩니다."
   - "단일글로 쓰면 정보가 압축돼 실용성이 떨어집니다."

4. 초안 결과는 단일 문자열이 아니라 배열로 저장한다.

```json
{
  "channel": "threads",
  "format_type": "short_thread",
  "recommended_length": 5,
  "why_this_format": "",
  "target_reader": "",
  "reader_problem": "",
  "posts": [
    {
      "index": 1,
      "role": "hook",
      "text": "",
      "media_note": ""
    },
    {
      "index": 2,
      "role": "step",
      "text": "",
      "media_note": ""
    }
  ],
  "review_note": "",
  "risk_flags": []
}
```

## 현재 관찰 메모

2026-05-19 기준 수동 관찰:
- AI 활용 고반응 게시글은 단일 팁보다 "사례/툴/리소스 묶음" 연결글에서 저장/공유 가치가 강했다.
- 40개 이상 mega thread는 조회수와 공유가 높지만, 기브니즈의 매일 운영 기본형으로는 무겁다.
- 네이버 플레이스/자영업 마케팅 쪽은 무료 자료, 질문형, 사장님 고민 공감형 게시글이 반응을 얻는다.
- 병원 마케팅은 개인 경험/업무 증거/전문성 기반 글이 더 맞다.
- 과한 수익 약속보다 "내 상황에 적용할 수 있는 작은 방법"이 기브니즈 톤에 맞다.
